[Coding Practice] 덧칠하기

덧칠하기 문제에 대한 설명은 프로그래머스 사이트를 참조바랍니다.

해당 문제는 원래 Arrays.copyOfRange()를 이용하여 덧칠한 section을 제거하는 로직으로 해결하려고 했었습니다. 그러나 계속 시간초과가 일어나서 Arrays.copyOfRange()를 주석처리 해놓고 실행했더니 적어도 시간이 초과하지는 않는 것을 발견하고 다른 방법을 찾았고, 굳이 배열의 요소를 제거하지 않고 index만 규칙에 맞게 옮기면서 덧칠 횟수를 찾는 로직을 다시 작성하였습니다.

import java.util.*;

class Solution {
    public int solution(int n, int m, int[] section) {
        // 최소값에서 m 만큼 더한 값 바로 전 값까지 배열에서 제거 후 answer++.
        // 그 뒤의 최소값에서 다시 m 만큼 더한 값의 바로 전 값까지 같은 방법 반복.
        // -> copyOfArray()의 시간이 너무 걸리는 문제.
        
        // -> index를 옮기며 횟수를 계산.
        int answer = 0;
        
        int i = 0;
        int j = 0; // 시작점을 0으로 해야 올바른 횟수가 계산된다.
        // 시작점을 section[0] + m으로 할 경우 마지막 부분이 제외된다.
        while (i < section.length) {
            if (section[i] < j) {}
            else if (section[i] >= j) {
                answer++;
                j = section[i] + m;
            }
            i++;
        }
        
        return answer;
    }
}

개인적으로는 while보다는 for문이 더 직관적으로 다가오는데, 다른 풀이 중에는 for문을 이용하여 조건에 맞지 않는 경우 continue로 다음 iteration으로 넘어가는 식의 로직도 볼 수 있었습니다.
