[Coding Practice] 카드 뭉치

카드 뭉치 문제에 대한 설명은 프로그래머스 사이트를 참조바랍니다.

본 문제는 단어들을 순서대로 사용해야하므로 Queue를 이용하여 해결하였습니다. 만들어야 하는 단어 배열인 goalQueue가 다 비어야 Yes라는 답이 나오는데, cardQueue1과 cardQueue2가 각각 먼저 다 비어버렸을 때의 상황도 역시 처리해줘야 합니다. 이 경우에는 아직 goalQueue가 다 없어지지 않았는데도 둘 중 한 쪽이 비었으므로 나머지 한 쪽과 goalQueue를 peek()해서 일치하지 않으면 바로 반복문을 탈출하도록 하였습니다.

import java.util.*;

class Solution {
    public String solution(String[] cards1, String[] cards2, String[] goal) {
        // 모든 배열을 Queue로 만든 뒤에 peek을 사용하여 goal과 값 비교,
        // 있으면 poll()로 제거.
        Queue<String> cardsQueue1 = new LinkedList<>();
        for (String card1 : cards1) {
            cardsQueue1.add(card1);
        }
        Queue<String> cardsQueue2 = new LinkedList<>();
        for (String card2 : cards2) {
            cardsQueue2.add(card2);
        }
        Queue<String> goalQueue = new LinkedList<>();
        for (String g : goal) {
            goalQueue.add(g);
        }
        
        String answer = "";
                
        while (true) {
            // 큐의 크기가 0일 경우
            if (goalQueue.size() == 0) {
                break;
            }
            
            // goalQueue 크기가 0이 아닐 때 두 가지 분기.
            if (goalQueue.size() > 0 && cardsQueue1.size() > 0
                && cardsQueue2.size() == 0
                && !cardsQueue1.peek().equals(goalQueue.peek())) {
                    break;
                }
            else if (goalQueue.size() > 0 && cardsQueue2.size() > 0
                && cardsQueue1.size() == 0
                && !cardsQueue2.peek().equals(goalQueue.peek())) {
                    break;
                }
            
            // 큐의 크기가 0보다 클 때
            if (cardsQueue1.size() > 0 && 
                cardsQueue1.peek().equals(goalQueue.peek())) {
                cardsQueue1.poll();
                goalQueue.poll();
            }
            else if (cardsQueue2.size() > 0 && 
                     cardsQueue2.peek().equals(goalQueue.peek())) {
                cardsQueue2.poll();
                goalQueue.poll();
            }
        }
        
        if (goalQueue.size() == 0) {
            answer = "Yes";
        }
        else {
            answer = "No";
        }
        
        // 복잡한 조건문을 쓰지 않고 goalQueue에서 poll()로 하나 뺀 뒤에 저장한 값을
        // cardsQueue1 및 cardsQueue2와 if - else if - else 문으로 비교하여 있으면 제거,
        // 두 큐에 다 존재하지 않으면 else 문으로 바로 가서 while문 탈출.
        return answer;
    }
}

코드 아래 부분 주석에서도 볼 수 있듯이 제 풀이에서는 약간 복잡한 조건문이 사용되었는데, 다른 풀이에서는 goalQueue에서 바로 poll()로 하나 뺀 뒤에 변수에 저장을 하고 cardsQueue1과 cardsQueue2와 비교하여 일치하는 단어가 있을 경우 바로 제거하며 두 Queue와 모두 일치하지 않으면 바로 반복문을 탈출하도록 더 간단하게 로직을 짠 것을 확인할 수 있었습니다.
