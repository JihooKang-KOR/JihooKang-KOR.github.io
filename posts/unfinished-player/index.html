[Coding Practice] 완주하지 못한 선수

완주하지 못한 선수 문제에 대한 설명은 프로그래머스 사이트를 참조바랍니다.

이 문제는 동명이인이 존재하는 경우도 고려해서 완주하지 못한 선수 한 명을 찾아내는 문제입니다. 처음에 시도했던 방법은 중복되는 요소를 없애기 위해 간단하게 HashSet으로 변경 후 순회 중에 값이 존재할 경우 해당 Set에서 그 값을 제거하는 식이었습니다. 그러나 이 방법은 동명이인이 3명 이상일 경우에는 2명째에 Set에서 값이 없어져버려 더 이상 확인이 불가능하다는 문제가 있었습니다. 그래서 두 번째 방법으로 HashMap을 사용하여 key에는 이름, value에는 이름의 등장 횟수를 저장하여 문제를 해결하였습니다.

import java.util.*;

class Solution {
    public String solution(String[] participant, String[] completion) {
        // 방법1 : completion 배열을 HashSet으로 변경 후,
        // participant 배열을 순회하여 값이 존재하지 않으면 반환.
        // 중복이 허용되므로 만약 HashSet에 값이 있을 경우 바로 해당 키를 삭제 처리.
        // -> 동명이인이 3명 이상일 경우 문제 해결 불가.
        
        // 방법2 : completion 배열을 HashMap으로 변경.
        // Map의 key는 이름, value는 이름 등장 횟수.
        // 모든 value를 1로 저장한 뒤, 순회 시에 이름이 등장할 때 마다 -1.
        // 이미 있는 key인 경우 1로 저장하지 않고 기존 값에 +1.
        // 바로 get()를 사용하여 0보다 작은 경우 return.
        String answer = "";
        
        HashMap<String, Integer> compMap = new HashMap<>();
        for (String comp : completion) {
            if (compMap.containsKey(comp)) {
                compMap.put(comp, compMap.get(comp) + 1);
            }
            else {
                compMap.put(comp, 1);
            }
            
        }
        
        for (String p : participant) {
            if (compMap.containsKey(p)) {
                compMap.put(p, compMap.get(p) - 1);
                // 동명이인의 경우
                if (compMap.get(p) < 0) {
                    answer = p;
                    break;
                }
            }
            // compMap에 아예 이름 자체가 없을 경우.
            else {
                answer = p;
                break;
            }
        }
        
        return answer;
    }
}

최종 결과 저장은 반복문의 사용 횟수를 줄이기 위해 등장 횟수를 저장하자마자 바로 조건문을 사용하여 진행하였습니다. 이렇게 코드를 만들었더니 효율성 테스트에서도 느리지 않은 시간으로 통과를 할 수 있었습니다.
